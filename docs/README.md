## 一图流总览

一次分析，数据是这样流动的——从你的输入，到一份带证据的答案：

```
                    你的问题  +  视频  +  (可选) 参考图
                                  │
                    ┌─────────────▼─────────────┐
                    │  ① 意图识别（自动判断）      │   关键词秒判 → 不行请AI → 兜底摘要
                    │  「你到底想干嘛？」          │
                    └─────────────┬─────────────┘
                                  │ 决定走下面哪条路
                    ┌─────────────▼─────────────┐
                    │  ② 预处理流水线             │   下载→抽帧→去重→转写→拼图
                    │  把视频拆成「盖了时间章的图」 │   （时间戳穿透全程，这是地基）
                    └─────────────┬─────────────┘
                                  │
     ┌──────────┬────────────────┼────────────────┬──────────────────┐
     ▼          ▼                ▼                ▼                  ▼
 ┌────────┐ ┌────────┐      ┌────────┐      ┌────────┐        ┌──────────┐
 │理解摘要│ │  定位  │      │  审核  │      │ 以图搜 │        │车牌/车辆 │
 ├────────┤ ├────────┤      ├────────┤      ├────────┤        ├──────────┤
 │看图+台词│ │粗扫圈范围│     │地毯式排查│     │像素预筛 │        │追车→攒帧 │
 │ AI看一次│ │→精扫到秒│     │低门槛高召回│   │→AI语义 │        │→按位投票 │
 │出结构摘要│ │找最早出现│     │→合并区间│     │  确认  │        │→AI兜底   │
 │        │ │        │      │        │      │→精扫   │        │→画框回写 │
 └────┬───┘ └───┬────┘      └───┬────┘      └───┬────┘        └────┬─────┘
      └─────────┴───────────────┼───────────────┴──────────────────┘
                                ▼
                    ③ 带证据的答案（answer.json）
                    时间戳 + 证据截图 + 置信度  (车牌另出标注视频 annotated.mp4)
```

## 六大功能速览

| 功能 | 一句话 | 核心套路 | 怎么触发 |
|---|---|---|---|
| **自动判断** | 自动挑最合适的功能 | 关键词秒判 → AI兜底 → 兜底摘要 | 直接打字提问 |
| **理解摘要** | 视频讲了啥的结构化总结 | 拼图+台词 → AI看一次 | "总结这个视频" |
| **定位** | 找东西最早在第几秒 | 粗扫圈范围 → 精扫到秒 | "红车最早出现在什么时间" |
| **审核** | 排查敏感内容，宁多勿漏 | 密采地毯排查 → 合并区间 | "有没有暴力画面" |
| **以图搜** | 拿图在视频里找 | 像素预筛 → 语义确认 → 精扫 | 附参考图 +「他在哪出现」 |
| **车牌/车辆** | 追车画框跟随 + 识别车牌 | 追车→攒帧→投票→AI兜底→回写 | "识别视频里的车牌" |

## 三条设计铁律（贯穿所有功能）

1. **诚实优先** —— 找不到就说找不到，并声明采样局限；绝不瞎编。
2. **永不崩溃** —— 每一步都有兜底，单点故障不会拖垮整个流程。
3. **换模型只改配置** —— Claude / GPT / GLM 可插拔，通常改一行配置、不动代码。

> 想从代码层面理解？内核在 `src/mingcha/`，唯一入口是 `orchestrator.py` 的 `Orchestrator.ask()`，
> 它串起「校验 → 分类 → 规划 → 预处理 → 分析 → 组装」整条流水线（详见项目根 `CLAUDE.md`）。

# 明察 MingCha —— 前后端分离 Web 应用启动指南

本地运行的前后端分离 Web 应用：浏览器上传视频/粘贴链接 → 选意图 → 看进度 → 看带截图证据的结果。
**不依赖 Docker、不依赖 nginx**。模型 API Key 由用户在界面自己填，只存浏览器本地与请求内存，不落盘。

对应设计文档：`docs/前后端分离-详细设计文档.md`。

## 目录

```
mingcha/
├── src/mingcha/       # 内核（已改造：progress 回调 / 请求级 Key 注入 / 结构化字段扩展）
├── backend/           # FastAPI 服务（REST + SSE + 产物托管 + SPA 托管）
│   ├── app/           # main / routers / task_manager / service / schemas / keys / security
│   └── tests/
├── frontend/          # React + Vite + TS SPA
│   ├── src/           # api / components / store / types
│   └── dist/          # 构建产物（生产由后端一并托管）
└── data/tasks/{id}/   # 运行时任务产物（gitignore）
```

## 一、前置依赖

- **Python 3.10+**（后端）
- **Node 18+**（仅在需要改/构建前端时；用发布包内置的 `frontend/dist` 可跳过）
- **ffmpeg / ffprobe**：推荐 `pip install static-ffmpeg`（免手装，后端启动自动注入 PATH）；
  或系统安装（`winget install ffmpeg` / `brew install ffmpeg` / `apt install ffmpeg`）。

## 二、启动后端

```bash
cd mingcha
python -m venv .venv && . .venv/bin/activate     # Windows: .venv\Scripts\activate
pip install -e . && pip install -e ./backend      # 装内核（根 pyproject）+ 后端
pip install static-ffmpeg "yt-dlp>=2024.1.0"      # ffmpeg 免手装 + 取视频
# 可选转写: pip install openai-whisper

uvicorn app.main:app --app-dir backend --port 8000
# 默认 --host 127.0.0.1，仅本机可达。局域网访问加 --host 0.0.0.0（注意 Key/产物会对同网段可达）
```

> 注：内核 `mingcha` 包由项目根 `pyproject.toml` 从 `src/` 构建，故用 `pip install -e .`。
> 后端 `pip install -e ./backend` 会自动拉起 fastapi / uvicorn / python-multipart。

## 三、启动前端

**A. 开发模式**（改前端时，热更新，`/api` `/artifacts` 自动代理到后端 8000）：
```bash
cd frontend && npm install && npm run dev        # http://localhost:5173
```

**B. 生产模式**（日常使用，推荐）：构建一次，后端一并托管，只开后端进程：
```bash
cd frontend && npm install && npm run build      # 产出 frontend/dist
# 之后只跑后端，浏览器开 http://localhost:8000
```

## 四、前端类型自动生成（可选，§8.9）

后端跑起来后，从 `/openapi.json` 生成 TS 类型，杜绝前后端契约漂移：
```bash
cd frontend && npm run gen:types                 # 写入 src/types/api.ts
```
（未生成时 `src/types.ts` 手写类型可保证正常开发/构建。）

## 五、环境变量（全部可选，§13.1）

| 变量 | 默认 | 说明 |
|---|---|---|
| `MINGCHA_DATA_DIR` | `./data` | 任务产物根目录 |
| `MINGCHA_MAX_CONCURRENCY` | `2` | 并发任务数（进程池大小） |
| `MINGCHA_TASK_TTL_HOURS` | `24` | 产物保留时长 |
| `MINGCHA_MAX_UPLOAD_MB` | `500` | 上传大小上限 |
| `MINGCHA_TASK_TIMEOUT_MIN` | `30` | 单任务超时 |
| `MINGCHA_FRONTEND_DIST` | `frontend/dist` | 前端构建产物目录 |
| `ANTHROPIC_API_KEY` / `OPENAI_API_KEY` / `ZHIPU_API_KEY` | 空 | 可选预置 Key（本地自用），优先级低于界面 Key |

## 六、Key 与安全

- Key 只经请求头 `X-Provider-Keys`（JSON），**不进 URL/query/日志/磁盘**；任务结束即释放。
- 优先级：**界面填的 Key > 后端环境变量**。
- 产物访问需 `task_token` + 路径穿越防护；默认绑回环 `127.0.0.1`。
- 产物默认 24h 后台清理。

## 七、测试

